# S01: Owned vfx.ron schema + appearance curve eval (tracer bullet)

**Goal:** Establish the owned, editor-ready per-Digimon `vfx.ron` schema and a pure, deterministic appearance curve evaluator as a vertical tracer bullet: a headless test loads `assets/digimon/agumon/vfx.ron` into a typed `VfxAsset` and evaluates scale/color keyframe curves deterministically (R004), and the Baby Flame impact fan-out renders from that data path windowed-side. Per D034 the schema is editor-ready (typed, introspectable verb params deriving Serialize+Deserialize+Reflect with deny_unknown_fields), so the future anim_graph+vfx GUI editor never forces a schema refactor. Placement-verb-in-Registry and full removal of VfxParticleKind/vfx_particle_kind are deliberately deferred to S02; this slice ports only the impact fan-out effect to prove the data path end-to-end.
**Demo:** Headless test loads assets/digimon/agumon/vfx.ron into a typed VfxAsset and evaluates appearance scale/color keyframe curves deterministically; Baby Flame impact fan-out renders from the data path in cargo winx.

## Must-Haves

- 1) A typed `VfxAsset` schema exists in the headless lib deriving Serialize+Deserialize+Reflect+Asset+TypePath with deny_unknown_fields, round-trips through RON, and rejects unknown fields. 2) Pure `eval_scale`/`eval_color` keyframe interpolation is deterministic and headless-tested at endpoints, midpoint, and before/after-range clamps plus empty/single-keyframe edge cases (R004). 3) `assets/digimon/agumon/vfx.ron` expresses the Baby Flame impact fan-out effect and a headless test loads it into `VfxAsset` and evaluates its curves to expected values. 4) The windowed Baby Flame impact burst (count/spread/ttl + per-tick scale/alpha/color) is driven by the asset via the pure resolver instead of hardcoded impact constants; `cargo build --features windowed` and the full headless `cargo test` stay green. 5) R012 (no numeric gameplay payload in the serialized SpawnParticle command) and R016 (no windowed deps leak into headless) invariants remain green.

## Proof Level

- This slice proves: Contract + integration. Headless contract (schema round-trip, deny_unknown_fields, deterministic curve eval, real-asset load) is proven by tests in the current message. Windowed integration is proven by `cargo build --features windowed` compiling the glue plus a windowed-feature-gated structural test exercising the asset→spawn-plan resolver. Real runtime required: yes (windowed build). Human/UAT: yes — visual confirmation of the impact fan-out via `cargo winx` is human-only (auto-mode cannot open a window, K001).

## Integration Closure

Upstream consumed: src/animation/anim_graph.rs (Command::SpawnParticle, ParticleId, VfxLocus/VfxMotion, Serialize/Deserialize+deny_unknown_fields idiom), src/animation/vfx.rs (VfxSpawnDescriptor/resolve_locus seam), src/combat/runtime/registry.rs (Registry<E>/iter pattern referenced for S02, not wired here), src/windowed/render.rs (spawn_baby_flame_impact_burst + impact-shard tick path). New wiring this slice: a new headless module src/animation/vfx_asset.rs exported from src/animation/mod.rs; windowed RonAssetPlugin::<VfxAsset> load of digimon/agumon/vfx.ron threaded into the impact spawn. Remaining before milestone is usable end-to-end: S02 hosts placement verbs in Registry<E> + generic render dispatcher and removes VfxParticleKind/vfx_particle_kind; S03 adds variant selection + ports Baby Burner detonate.

## Verification

- Windowed asset load must fail visibly: if digimon/agumon/vfx.ron fails to parse or the requested effect id is absent, render.rs logs a structured warn! on target "windowed.agumon_playback" (effect id + reason), mirroring the existing missing-graph diagnostic, and falls back to the prior hardcoded impact path rather than silently dropping VFX. deny_unknown_fields gives a precise load-time parse error naming the offending field.

- [x] **T02: Implement pure deterministic appearance curve evaluator + effect-appearance resolver** `est:1h`
  Why: R004 requires all appearance verb math to be pure and headless-testable; keeping the curve eval and asset->spawn-params resolution in the headless lib lets the windowed render path (T04) be thin glue rather than untested rendering logic. Do: in src/animation/vfx_asset.rs add pure functions: `eval_scale(curve: &ScaleCurve, progress: f32) -> f32` and `eval_color(curve: &ColorCurve, progress: f32) -> [f32; 4]`, each performing piecewise-linear interpolation between keyframes sorted by `t`, clamping `progress` to [0,1], returning the first keyframe value for progress before the first `t`, the last for progress after the last `t`, and a sensible default (1.0 scale / opaque white) for an empty curve. Add `resolve_effect<'a>(asset: &'a VfxAsset, effect_id: &str) -> Option<&'a EffectDef>` and a small pure `ImpactSpawnPlan { count: u32, spread_px: f32, ttl_ticks: u32 }` constructor `spawn_plan(effect: &EffectDef) -> ImpactSpawnPlan` reading from Appearance, so the windowed layer asks the lib for concrete spawn parameters. All functions are Bevy-render-free (may use bevy math/Reflect already in the lib). Write tests/animation/vfx_asset_eval.rs and register it in tests/animation.rs covering (Q7 negative/boundary): eval at progress 0.0 and 1.0 equals the endpoint keyframes; eval at a midpoint equals the linear interpolant; progress clamps below 0 and above 1; empty curve returns the documented default; single-keyframe curve returns that keyframe for all progress; eval is deterministic (same input -> identical output across repeated calls). Done when: vfx_asset_eval tests pass and the functions are pure (no I/O, no Bevy world/render types). Requirement impact (Q4): exercises validated R004 determinism; no contract broken.
