# S01: Enoki as sole VFX renderer

**Goal:** Make bevy_enoki the sole particle VFX renderer for every Agumon effect and delete the custom quad VFX system. After this slice: in `cargo run --features windowed`, the full Baby Flame sequence (charge buildup, ember swirl, traveling projectile, impact) plus Sharp Claws and Baby Burner all render through enoki one-shots / emitters; the quad system (advance_vfx_particles, the quad spawn loop, VfxParticle/Target/Source, VfxVisuals, the windowed AgumonVfx/vfx.ron loader) is gone; `cargo build --features windowed` is green; full `cargo test` is green; and the dependency-gating test still proves bevy_enoki never leaks into the headless graph. VFX visual quality is K001 manual (deferred — auto-mode verifies via source-contract tests + build/test green).
**Demo:** In cargo winx, the full Baby Flame sequence (charge, ember, projectile, impact) plus Sharp Claws and Baby Burner all render through enoki; the quad system code is gone; cargo build --features windowed is green and the dep-gating test still passes. (VFX quality is K001 manual.)

## Must-Haves

- All six Agumon effect ids (baby_flame.charge/ember/projectile/impact, baby_burner.detonate, sharp_claws.slash) route through enoki; no quad rendering code remains in src/windowed/render.rs (no advance_vfx_particles, no `for i in 0..count` quad loop, no VfxParticle/VfxParticleTarget/VfxParticleSource, no VfxVisuals/load_vfx_visuals/vfx_texture_handle, no windowed AgumonVfx/load_agumon_vfx/diagnose_agumon_vfx_load); charge/ember clear at launch and the projectile chains the impact burst via the new enoki lifecycle layer; lib VfxAsset/resolve_effect survive (headless + data-contract tests unchanged); `cargo build`, `cargo build --features windowed`, `cargo test`, and `cargo test --test dependency_gating` all green.
