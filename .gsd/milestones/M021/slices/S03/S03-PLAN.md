# S03: Mode parity (DryRun ≡ Execute ≡ Preview) + Two-clock invariant

**Goal:** Prove three invariants on the S02 BeatRunner/SkillCtx/Clock primitives: (I2/D024) DryRun ≡ Execute ≡ Preview produce a byte-identical Intent stream on a branched timeline; (I3/D026) Clock::HeadlessAuto ≡ Clock::Windowed produce the same end-of-cast Intent stream (only timing differs — Windowed stalls on a Presentation-bearing beat); (F4) a Loop whose exit_when never fires halts at MAX_HOPS=256 with an observable log signal instead of hanging. Achieved via one contained runner edit (clock-aware stepping + AwaitingCue/resume_cue + circuit-breaker log::warn!) and three deterministic integration tests. Scope per D005/D006/D007: stream-parity only (no applier change), edge-predicate branching (no SelectorCtx change), minimal Windowed latch (full Windowed runtime is S04).
**Demo:** Test DryRun≡Execute≡Preview verde su chain ramificata; two-clock verde; circuit breaker @256.
