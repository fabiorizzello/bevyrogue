The live windowed soak run is integration-level and display-dependent; where no display is available the aggregator/threshold logic remains fully unit-proven and the limitation is documented in the comparison artifact rather than faked.

No new runtime wiring crosses subsystem boundaries except the frame-time aggregator, which is fed Time::delta_secs only inside the existing windowed_validation_tick (Clock::Windowed presentation layer) and emitted alongside the existing validation_snapshot line — it does not touch src/combat resolution. The boundary map and closeout docs are pure documentation over already-wired seams; the registry/VFX tests exercise existing lib APIs. R004 determinism preserved (no wall-clock/RNG in the resolution stream).
